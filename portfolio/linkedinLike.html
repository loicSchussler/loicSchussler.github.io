Projet réalisé en groupe de 4.

Rest API developpé avec Asp.Net (C#) qui imite un site de mise en relations entre recruteurs et candidats comme Linkedin.

Outils informatiques utilisés

  • Langage C# avec le framework ASP.Net Core MVC
  • Visual Studio 2019 en tant qu'environnement de développement.
  • MySQL Workbench pour gérer la base de données.
  • Base de données heroku
  • Swagger pour avoir une interface décrivant la rest API<./li>
  • Git pour collaborer et mettre à jour le code du projet.
  • Discord pour communiquer au sein de l'équipe.
  • Trello pour se répartir les tâches.

Cahier des charges du projet

Objectif global

Créer une REST api prête à une implémentation sur une structure frontEnd de mise en relations de candidats et d'employeurs à la façon de la très connue plateforme Linkedin.

Fonctionnalités requises

  1. Les principales méthodes GET,POST,PUT,DELETE permettant de manipuler les utilisateurs et les informations associées dans une base de données (profile,compétences,disponibilité...).
  2. L'API doit avoir des fonctionnalités (EndPoint) publics et privées.
  3. Le système est capable d'enregistrer des utilisateurs avec un role différent (candidat ou employeur) et permettre ensuite la connexion à leur profil.
  4. Possibilité de mettre en avant certains candidats auprès d'employeurs (option payante de visibilité).
  5. Les utilisateurs peuvent voir les profiles des autres (leurs compétences,les jobs réalisés, les avis) et particulèrement les employeurs peuvent voir les profiles des candidats et leur envoyer un message pour les recruter (système de messagerie).

Mon travail

Résumé

Ce projet a été pour moi, un projet très intéréssant car en le réalisant j'ai appris de nombreuses choses sur le backend avec AspNet Core. Mes missions principales ont été de réaliser la structure de la base de données, les fonctionnalités liés au profil des candidats comme le système de note et d'avis.

Fonctionnalités liées au profil des candidats

Pour coder ces fonctionnalités il a été nécessaire de créer 2 controlleurs d'api, un pour les candidats et un pour les employeurs. Les controlleurs disposent de methodes permettant d'avoir le profil d'un candidat ou employeur en particulier ou alors la liste de tous. Le profil retourné sous forme de fichier json lors d'une requête get contient les informations sur l'identité de la personne ainsi que son activité sur la plateforme. J'ai aussi ajouter une méthode put permettant aux employeurs de noter sur 10 la mission réalisée.
